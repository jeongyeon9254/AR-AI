테스트 코드를 작성합니다. 다음 절차를 따르세요:

1. 대상 코드를 분석하여 테스트 시나리오를 설계합니다
2. 정상 케이스 (happy path) 테스트를 작성합니다
3. 엣지 케이스 및 에러 케이스를 작성합니다
4. 기존 테스트 패턴 및 프레임워크에 맞춰 작성합니다
5. 모킹이 필요한 외부 의존성을 식별합니다

테스트 파일은 기존 프로젝트 구조와 컨벤션을 따릅니다.
한국어 주석을 포함합니다.

## E2E 테스트 작성 가이드

목표: Playwright 기반 E2E 테스트코드 작성

### 방식

1. **코드 분석**: `page > [page].component` 파일과 해당 컴포넌트의 하위 컴포넌트를 참고한다.
2. **API 인터페이스 재사용**: API 응답값 확인 시 해당 페이지에서 사용하는 API의 인터페이스를 재사용한다.
3. **POM 우선 작성**: Page Object Model(POM)을 먼저 만들고, 그 이후 테스트코드를 작성한다.
4. **파일 위치**: `alpha-review > src > test > e2e > review-list > [page]` 폴더에 작성한다.
5. **TC 기반 작성**: 테스트 코드는 아래 TC 형식을 기반으로 작성하며 styleguide에 따른다.
6. **넘버링**: 테스트는 `##` 단위로 하며 `TC-00` 넘버링을 붙인다.
7. **import**: `import { expect, test } from '../../../util/authenticated-test';` 를 사용한다.
8. **goto**: `/dashboard/{route}` 경로로 진행한다.
9. **주석**: 테스트 케이스 작성 시 `##` 단위로 작성된 테스트코드 위에 주석으로 테스트 케이스의 원본을 요약해서 서술한다. 이때 특정 id 같은 값은 제거한다.

### TC 작성 형식

```
## 테스트전 공통사항

- 테스트 진행전 공통사항에 대한 서술

## TC-01 테스트명

- 테스트 플로우 1
- 테스트 플로우 2
- 테스트 플로우 3
```

### 예시 코드 구조

```typescript
import { expect, test } from '../../../util/authenticated-test';

// 테스트전 공통사항
// - 대시보드 페이지로 이동
// - 초기 데이터 로드 대기
test.beforeEach(async ({ page }) => {
  await page.goto('/dashboard/{route}');
  // 공통 전처리
});

// TC-01 테스트명
// - 테스트 플로우 1 요약
// - 테스트 플로우 2 요약
// - 테스트 플로우 3 요약
test('TC-01 테스트명', async ({ page }) => {
  // 테스트 구현
});
```
